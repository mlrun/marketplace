validate_great_expectations.validate_great_expectations.validate_expectations(context: mlrun.execution.MLClientCtx, data: mlrun.datastore.base.DataItem, expectation_suite_name: str, data_asset_name: str, datasource_name: str = 'pandas_datasource', data_connector_name: str = 'default_runtime_data_connector_name', datasource_config: Optional[dict] = None, batch_identifiers: Optional[dict] = None, root_directory: Optional[str] = None, checkpoint_name: Optional[str] = None, checkpoint_config: Optional[dict] = None) → None[source]#

Main function to validate an input dataset, datasource, data connector, and expectation suite.

Runs the Great Expectation validation and logs whether the validation was a success as well as the output page of the data docs.

Parameters

  • context – MLRun context.

  • data – Data to validate. Can be local or remote link.

  • expectation_suite_name – Name of expectation suite to validate against.

  • data_asset_name – Name of dataset in Great Expectations.

  • datasource_name – Name of datasource to use for validation.

  • data_connector_name – Name of data connector to use for validation.

  • datasource_config – Full configuration for datasource. For use with custom data sources other than the default pandas datasource.

  • batch_identifiers – Custom metadata for identifying particular batches of data. For use when not using the default batch identifiers.

  • root_directory – Path to underlying Great Expectations project. Defaults to MLRun project artifact path if not specified.

  • checkpoint_name – Name of checkpoint to use for validation.

  • checkpoint_config – Full configuration for checkpoint. For use with custome checkpoint config other than the default.
